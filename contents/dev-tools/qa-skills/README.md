# Software QA Skills with AI CLIs

This directory contains guides and skills for performing Software Quality Assurance (QA) tasks using AI-powered command-line interfaces like **Kimi CLI** and **Claude Code CLI**.

## Overview

AI models can significantly accelerate QA workflows by automating repetitive tasks, generating comprehensive test plans, and identifying edge cases that might be missed by manual testers.

## Available Skills

- **[Test Case Design](test-case-design.md)**: Strategies and prompts for designing robust test cases.
- **[Kimi QA Skills](kimi-qa-skills.md)**: Leveraging Kimi's long-context capabilities for deep code analysis and QA.
- **[Claude QA Skills](claude-qa-skills.md)**: Using Claude's reasoning power for complex test scenario generation.

## How to Use These Skills

1. **Setup your CLI environment**: Ensure you have [Kimi CLI](../cli/kimi-cli/README.md) or [Claude Code CLI](../cli/claude-code-cli/README.md) installed.
2. **Choose a task**: Select a QA task like "Test Case Design" or "Bug Reproduction".
3. **Execute prompts/tools**: Use the recommended commands and prompts provided in each skill guide.
4. **Refine results**: Review the AI-generated output and refine it based on your project's specific requirements.

## Key Benefits

- **Speed**: Rapidly generate test cases and documentation.
- **Coverage**: Identify more edge cases and boundary conditions.
- **Consistency**: Maintain a standardized format for QA artifacts.
- **Integration**: Easily integrate QA tasks into your development terminal workflow.
