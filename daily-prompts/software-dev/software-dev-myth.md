# Software Myth

## 1. Tester Pointer of View

一个资深测试工程师的角度看Dev，产品，项目经理，从一个不在大厂的工程师这些职位上的人有很多明显的缺陷:
1. Dev, 过于认为自己是做技术的，不愿意接触业务，但是忽视了业务也是逻辑和代码没有本质区别，没有服务心态，制造了太多黑箱并且喜欢自己特殊化而不遵守流程和沟通原则，非常不善于用平白的话描述自己做的事情，某种程度上就是从来没有考虑过听众的感觉
2. 产品，对于底层一点技术实现的不了解让他无法掌握需求的颗粒度，要么非常粗糙，要么过于细致而印象开发的横向架构思维，自视过高，但是没有自己完整的思考框架，为了什么，怎么样统一的产品逻辑，用客户要求或者老板要求搪塞自己需要的过滤和是否需要用新的解决办法而不是别人需要什么功能就做什么功能
3. 项目经理，同样的自视过高，其实什么都做不了，还不如资深测试做项目经理，首先他不能保证质量和交付时间，其次他带来的不不要的压力，同时他不能为真正项目瓶颈说话，对测试及其不友好，个人感觉添乱的时候多过有用的时候，大部分的时间不知道在做什么，管不了预算，管不了人也不能了解项目难点，只能压测试进度为了完成任务，同时对于强硬的开发不敢怎么样，实际上根本不能解决进度问题，实际出现的问题都在下游只是他没能力也没胆量解决甚至提问题，一个完全被夸大的职位。

请AI对这些点评做客观和全面评价。
